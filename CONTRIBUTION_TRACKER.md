## Weekly Checklist

### Week 1
- [ ] Choose 3 projects to focus on
- [ ] Read CONTRIBUTING.md for each project
- [ ] Set up local development environment
- [ ] Find 5 "good first issue" tickets
- [ ] Make first documentation PR

### Week 2
- [ ] Submit 2-3 documentation PRs
- [ ] Comment on 3 existing issues
- [ ] Review 2 other people's PRs
- [ ] Create 1 quality issue

### Week 3
- [ ] Fix 1-2 "good first issue" bugs
- [ ] Submit PRs with tests
- [ ] Respond to all reviewer feedback
- [ ] Help triage 2 new issues

### Week 4
- [ ] Maintain 1-2 PRs per week
- [ ] Create 1-2 issues
- [ ] Review progress and adjust strategy
- [ ] Plan for next month
